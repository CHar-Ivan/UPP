# Код создания БД и работы в отключенном режиме

## 📋 Общая схема работы

```
MainWindow.cs
    ↓
InitializeDatabase()
    ↓
Database.EnsureCreated()  ← Создание БД
    ↓
DatabaseInitializer.InitializeWithSampleData()  ← Заполнение примерами
    ↓
LoadAllData()  ← Загрузка в UI
```

## 1️⃣ **Создание контекста БД** (`LibraryDbContext.cs`)

### Строка подключения (ОТКЛЮЧЕННЫЙ РЕЖИМ):
```csharp
protected override void OnConfiguring(DbContextOptionsBuilder optionsBuilder)
{
    // LocalDB - локальный экземпляр SQL Server Express (не требует сервера)
    // Работает полностью offline без интернета
    string connectionString = @"Server=(LocalDB)\MSSQLLocalDB;Database=LibraryDB;Integrated Security=true;";
    optionsBuilder.UseSqlServer(connectionString);
}
```

**Почему это автономный режим:**
- `(LocalDB)\MSSQLLocalDB` - локальный экземпляр на компьютере пользователя
- Не требует запущенного SQL Server сервера
- БД хранится локально: `C:\Users\[Username]\AppData\Local\Microsoft\Microsoft SQL Server Local DB\`
- Работает полностью offline

### Конфигурация таблиц:
```csharp
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    // Таблица Authors
    modelBuilder.Entity<Author>()
        .HasKey(a => a.Id);  // Primary Key
    modelBuilder.Entity<Author>()
        .Property(a => a.Name)
        .IsRequired()  // NOT NULL
        .HasMaxLength(200);
    
    // Связь 1:∞ (один автор - много книг)
    modelBuilder.Entity<Book>()
        .HasOne(b => b.Author)
        .WithMany(a => a.Books)
        .HasForeignKey(b => b.AuthorId)
        .OnDelete(DeleteBehavior.Cascade);  // При удалении автора удаляются его книги
    
    // Связь ∞:∞ через таблицу Borrowings
    modelBuilder.Entity<Borrowing>()
        .HasOne(br => br.Member)
        .WithMany(m => m.Borrowings)
        .HasForeignKey(br => br.MemberId)
        .OnDelete(DeleteBehavior.Cascade);
}
```

## 2️⃣ **Инициализация БД** (`MainWindow.xaml.cs`)

### Создание БД при первом запуске:
```csharp
private void InitializeDatabase()
{
    try
    {
        // 1. EnsureCreated() - создает БД если её нет
        // Выполняет DDL (CREATE TABLE) скрипты
        // В режиме offline - работает локально
        _dbContext.Database.EnsureCreated();
        
        // 2. Заполняем примерами данных
        DatabaseInitializer.InitializeWithSampleData(_dbContext);
        
        MessageBox.Show("База данных инициализирована успешно!", "Успех", 
                        MessageBoxButton.OK, MessageBoxImage.Information);
    }
    catch (Exception ex)
    {
        MessageBox.Show($"Ошибка при инициализации БД: {ex.Message}", 
                        "Ошибка", MessageBoxButton.OK, MessageBoxImage.Error);
    }
}
```

**Что происходит под капотом:**
1. `EnsureCreated()` проверяет существует ли БД `LibraryDB`
2. Если нет - создает все таблицы (Authors, Books, Members, Borrowings)
3. Все это выполняется на локальном SQL Server Express
4. Результат: БД создана локально на компьютере

## 4️⃣ **CRUD операции в отключенном режиме**

### ➕ Добавление (CREATE):
```csharp
private void AddAuthor_Click(object? sender, RoutedEventArgs? e)
{
    var author = new Author
    {
        Name = AuthorNameBox.Text,
        Nationality = AuthorNationalityBox.Text,
        Email = AuthorEmailBox.Text,
        BirthDate = AuthorBirthDatePicker.SelectedDate ?? DateTime.Now
    };

    _dbContext.Authors.Add(author);        // Добавить в контекст
    _dbContext.SaveChanges();              // Сохранить в локальную БД
    
    RefreshAuthors_Click(null, null);      // Обновить UI
}
```

### 📖 Чтение (READ):
```csharp
private void RefreshAuthors_Click(object? sender, RoutedEventArgs? e)
{
    // Загружаем данные ИЗ локальной БД
    var authors = _dbContext.Authors.ToList();
    AuthorsDataGrid.ItemsSource = authors;  // Привязка к UI

```

### Чтение с связанными данными:
```csharp
private void RefreshBooks_Click(object? sender, RoutedEventArgs? e)
{
    // Include() - загружает связанные данные (Автора для каждой книги)
    var books = _dbContext.Books.Include(b => b.Author).ToList();
    BooksDataGrid.ItemsSource = books;
}
```

### ✏️ Обновление (UPDATE):
```csharp
private void ReturnBook_Click(object? sender, RoutedEventArgs? e)
{
    var selected = BorrowingsDataGrid.SelectedItem as Borrowing;
    if (selected == null) return;
    
    selected.ReturnDate = DateTime.Now;    // Изменить данные
    _dbContext.SaveChanges();              // Сохранить изменения в БД
    RefreshBorrowings_Click(null, null);
}
```

### ❌ Удаление (DELETE):
```csharp
private void DeleteAuthor_Click(object? sender, RoutedEventArgs? e)
{
    var selected = AuthorsDataGrid.SelectedItem as Author;
    if (selected == null) return;
    
    _dbContext.Authors.Remove(selected);   // Удалить из контекста
    _dbContext.SaveChanges();              // Сохранить удаление в БД
    
    // OnDelete(DeleteBehavior.Cascade) автоматически удалит его книги!
    RefreshAuthors_Click(null, null);
}
```

## 5️⃣ **Автономность приложения**

### Почему работает offline:

```csharp
// 1. SQL Server работает локально
string connectionString = @"Server=(LocalDB)\MSSQLLocalDB;Database=LibraryDB;...";

// 2. Все данные хранятся на компьютере
// Нет обращений к интернету

// 3. EnsureCreated() создает схему локально
_dbContext.Database.EnsureCreated();

// 4. SaveChanges() сохраняет в локальную БД
_dbContext.SaveChanges();

// 5. Приложение закрывает соединение
protected override void OnClosed(EventArgs e)
{
    _dbContext?.Dispose();  // Освобождаем ресурсы
    base.OnClosed(e);
}
```

### Результат:
- ✅ Работает без интернета
- ✅ БД остается на компьютере пользователя
- ✅ Данные сохраняются между запусками
- ✅ Полная конфиденциальность (не отправляется никуда)

## 7️⃣ **Связи между таблицами (Foreign Keys)**

### Создание связей в коде:
```csharp
// Связь Book → Author (много к одному)
modelBuilder.Entity<Book>()
    .HasOne(b => b.Author)           // У Book ОДИН автор
    .WithMany(a => a.Books)          // У Author МНОГО книг
    .HasForeignKey(b => b.AuthorId)  // Внешний ключ - BookAuthorId
    .OnDelete(DeleteBehavior.Cascade); // При удалении автора удаляются его книги

// В БД создается:
// CREATE TABLE Books (
//     Id INT PRIMARY KEY,
//     Title NVARCHAR(300) NOT NULL,
//     AuthorId INT FOREIGN KEY REFERENCES Authors(Id)
//         ON DELETE CASCADE
// )
```

### Использование связей:
```csharp
// Загружаем книги с информацией об авторах
var books = _dbContext.Books
    .Include(b => b.Author)  // Получаем связанного автора
    .ToList();

// Вывод: у каждой книги есть автор (Author.Name)
foreach (var book in books)
{
    Console.WriteLine($"{book.Title} - {book.Author.Name}");
}
```

## 📊 SQL запросы, которые выполняются

### Создание таблицы (выполняется один раз):
```sql
CREATE TABLE Authors (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Name NVARCHAR(200) NOT NULL,
    Nationality NVARCHAR(100),
    BirthDate DATETIME,
    Email NVARCHAR(100)
);

CREATE TABLE Books (
    Id INT PRIMARY KEY IDENTITY(1,1),
    Title NVARCHAR(300) NOT NULL,
    ISBN NVARCHAR(20),
    Genre NVARCHAR(100),
    PublishDate DATETIME,
    AuthorId INT FOREIGN KEY REFERENCES Authors(Id) ON DELETE CASCADE
);
```

### Добавление данных:
```sql
INSERT INTO Authors (Name, Nationality, BirthDate, Email)
VALUES ('Лев Толстой', 'Россия', '1828-09-09', 'tolstoy@mail.ru');

INSERT INTO Books (Title, ISBN, Genre, PublishDate, AuthorId)
VALUES ('Война и мир', '978-5-17-977142-5', 'Исторический роман', '1869-01-01', 1);
```

### Загрузка с JOIN:
```sql
SELECT b.*, a.Name as AuthorName
FROM Books b
INNER JOIN Authors a ON b.AuthorId = a.Id;
```
