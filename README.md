# ЖЭК Management System - WPF Application

## Описание

Приложение для управления Жилищно-Коммунальным Хозяйством (ЖЭК). Работает в режиме **offline-first** с синхронизацией SQL Server.

## Возможности

✅ **Управление домами** - добавление, удаление, просмотр жилых зданий  
✅ **Управление квартирами** - добавление, удаление, просмотр квартир  
✅ **Управление услугами** - создание видов коммунальных услуг  
✅ **Управление заявками** - создание и отслеживание заявок на обслуживание  
✅ **Русский язык** - полная поддержка кириллицы  
✅ **Offline режим** - работает без интернета с sample данными  
✅ **SQL Server синхронизация** - подключение и синхронизация данных  

## Требования

- .NET 8.0 SDK
- Windows (WPF требует Windows)
- SQL Server (опционально, для синхронизации)

## Использование

### Работа в Offline режиме (по умолчанию)

1. Запустите приложение
2. Приложение загружает sample данные (2 дома, 3 квартиры, 3 услуги, 2 заявки)
3. Вы можете добавлять новые данные, все хранится в памяти
4. При закрытии приложения все данные теряются

### Подключение к SQL Server

#### Подготовка базы данных

1. Откройте SQL Server Management Studio
2. Выполните скрипт `CREATE_DATABASE.sql`:

```sql
-- Будут созданы таблицы для системы ЖЭК
```

#### Подключение через приложение

1. Убедитесь, что SQL Server запущен на `DESKTOP-DVM8L4A` (или отредактируйте строку подключения в коде)
2. Нажмите кнопку **"🔗 Подключиться и синхронизировать"**
3. Приложение загрузит все данные с сервера
4. Теперь все операции (добавление, удаление) будут сохраняться на SQL Server

```
Приложение (WPF)
    ├── Offline режим: In-memory кэш (быстро, но теряется при закрытии)
```

### Компоненты

- **MainWindow.xaml.cs** - главное окно и UI логика
- **Data/JekDatabaseService.cs** - сервис управления данными
- **Data/JekDbContext.cs** - Entity Framework контекст
- **Models/House.cs** - модели данных (House, Apartment, MaintenanceType, etc.)

### Модели данных

#### House (Дом)
- Id, Address, BuildingNumber, FloorsCount, ApartmentsCount, BuildYear, Material

#### Apartment (Квартира)
- Id, HouseId, ApartmentNumber, Floor, Area, RoomsCount, OwnerName, OwnerPhone, OwnerEmail, MoveInDate

#### MaintenanceType (Вид услуги)
- Id, Name, Description, Unit, TariffPerUnit, IsActive

#### MaintenanceRequest (Заявка на обслуживание)
- Id, ApartmentId, MaintenanceTypeId, Description, Status, Priority, CreatedDate, etc.

#### Utility (Показания услуг)
- Id, ApartmentId, MaintenanceTypeId, PreviousReading, CurrentReading, Amount, etc.

#### MaintenanceTask (Работы по обслуживанию)
- Id, HouseId, MaintenanceTypeId, Title, Description, Status, etc.

## Строка подключения SQL Server

Текущая конфигурация использует:

```
Server=DESKTOP-DVM8L4A;Database=JEK_DB;Integrated Security=true;Encrypt=false;TrustServerCertificate=true;
```

Для изменения отредактируйте `Data/JekDbContext.cs` метод `OnConfiguring()`:

```csharp
string connectionString = @"Server=YOUR_SERVER;Database=JEK_DB;Integrated Security=true;Encrypt=false;TrustServerCertificate=true;";
optionsBuilder.UseSqlServer(connectionString);
```

## Решение проблем

### Проблема: Приложение не подключается к SQL Server
**Решение:** 
- Проверьте имя сервера (DESKTOP-DVM8L4A)
- Убедитесь, что SQL Server запущен
- Проверьте логи в файле `jek_sync_log.txt`

### Проблема: Русский текст показывается неправильно
**Решение:** 
- UTF-8 кодировка уже настроена в проекте
- Перезагрузите приложение

### Проблема: Данные исчезают после закрытия приложения
**Это нормальное поведение** - приложение работает в offline режиме с in-memory кэшем. Данные не сохраняются на диск. Для сохранения нажмите "Подключиться и синхронизировать" чтобы загрузить/сохранить данные на SQL Server.

## Как использовать

### 1. Запуск приложения
- Откройте проект в Visual Studio
- Нажмите F5 или кнопку "Start" для запуска приложения

### 2. Инициализация БД
- При первом запуске приложение автоматически создаст базу данных SQL Server
- Будут добавлены примеры данных для демонстрации
- Требуется установленный SQL Server Express или LocalDB

### 3. Работа с данными

#### Добавление авторов
1. Перейдите на вкладку "Авторы"
2. Заполните поля: Имя, Национальность, Email, Дата рождения
3. Нажмите "Добавить автора"

#### Добавление книг
1. Перейдите на вкладку "Книги"
2. Заполните: Название, ISBN, Жанр, Дата публикации
3. Выберите автора из выпадающего списка
4. Нажмите "Добавить книгу"

#### Добавление членов библиотеки
1. Перейдите на вкладку "Члены библиотеки"
2. Заполните: Имя, Email, Телефон
3. Нажмите "Добавить члена"

#### Создание заимствования
1. Перейдите на вкладку "Заимствования"
2. Выберите члена и книгу
3. Нажмите "Создать заимствование"
4. Для отметки возврата: выберите запись и нажмите "Отметить как возвращено"
